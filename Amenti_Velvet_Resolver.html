VELVET RESOLVERThe Migration of Souls

One Thousand Dead · Six Opinions · One Table

It began as housekeeping — move the character cards into one file — and turned out to be nothing of the kind. There were not too many files. There was no arbiter: six separate pieces of code each held a private opinion about whether two names meant the same human, and they disagreed one hundred and ninety-one times. The repair renamed nothing, deleted nothing anyone had made, and never took the ship down on purpose. Hence velvet.

NOT ONE CANONICAL KEY WAS CHANGED.
THE REVOLUTION HAPPENED IN THE LOOKUP, NOT IN THE DATA.

1 · The wrong diagnosis, twice

The brief was to consolidate. The first instinct was to move everything into Supabase and retire the Google Sheet. The captain pushed back — doesn't the CSV feed other parts of the application? — and the pushback was right and the instinct was wrong.

The ledger carries twelve columns, not a name list: title, dates, biography, links, region, location, and gender, dialect and voice, which are what library.js folds into the speech style string. Retiring it would have silently cast a thousand figures to the neutral voice. It is also read by properties outside this repository. The correction was not to move the Sheet but to stop treating it as a runtime rival — and that reframing survived the night.

The second wrong turn was mine and smaller: the alias table was first drawn with the ENGINE’s key form as canonical. The topic forensics then counted it — 31 topic ids embed the short record key, 27 a surname form, and exactly ONE the engine’s. Canonical was already decided; it just had not been counted.

2 · What was actually wrong

Six pieces of code answered the question are these the same person? and none of them agreed:

Underneath them all sat a ledger with ninety-four people entered twice — same name, same source link, different rank. Two ranked lists had been merged and the overlap never reconciled.

Figure 1 · the fault, and the shape of the repair.

THE MATCHER WAS NOT DANGEROUS BECAUSE IT FAILED.
IT WAS DANGEROUS BECAUSE IT ANSWERED.

Every one of those was confident, plausible, and wrong, and the arena looked perfectly well the entire time. Nothing was visibly broken, so nothing was ever investigated. The matcher had been written to be careful — its own comment named the Marcus Manlius trap it was built to prevent, and it did prevent that one. It caused seventeen others.

3 · The instruments

Nothing was moved until it had been measured. Four probes were written, each read-only, each emitting a plain text report:

probe 1cross-referenced every source at once. Found 0 of 53 quiz figures joining a character by exact key, and 94 duplicated ledger people.
probe 2tested the matcher against every person on the page, not just the ones with quizzes. Found 191 of 319 wrong.
ledger checkwritten after too many console snippets. One tag, one file, every ledger question answered in a single pass.
resolvera dry run that simulated the migration and reported the diff before anything was wired.
Probe two’s own four-artifact audit reported Caesar and Gandhi as having no quiz. They plainly do. The instrument had matched by norm(name), so “GAIUS JULIUS CAESAR” never met “Julius Caesar” — it fell into the exact trap it was built to measure. Worth recording, because an instrument that shares the system’s assumptions will confirm them.

5 · Three failures worth keeping

A · The silent parse

config.js was edited correctly and repeatedly, and nothing happened. A missing closing quote meant the file threw on parse, so AMENTI_CONFIG was never defined at all — and the browser kept serving the last version that had parsed. The site therefore worked, on stale settings, while every fresh edit appeared to do nothing.

A CONFIG THAT FAILS TO PARSE DOES NOT LOSE ONE SETTING.
IT LOSES ALL OF THEM, QUIETLY, AND KEEPS RUNNING ON THE OLD ONES.

B · The silent fallback

When the ledger 400’d, Page1 fell back to ./names.csv exactly as written. A thousand figures became thirty-eight and the page carried on without a word. The glass gate held — but a gate that degrades in silence is only half a gate. A fallback should announce itself.

C · The instrument that shared the bug

Recorded above. A probe written by the same hand, with the same assumptions, will confirm those assumptions. The fix was adversarial testing: seventeen pairs taken from real wrong answers, asserted on every run.

6 · The repair

Exact match, or null. Four forms of a name are indexed — raw, lowercased, slugified, normalised — and if none hits, the answer is unknown and the caller decides. There is no last-name fallback and no containment. Where two records claim one spelling, rank decides: seed > rich > thin, so a hand-written decision beats a real character, and a real character beats a bare name on a list.

The canonical key is the one already in use. Nothing was renamed, so art keys, topic ids, reading rooms, group maps, bookmarked URLs and every recorded seeker progress row kept working untouched. That is the velvet. A migration that renames is a migration that breaks bookmarks and orphans progress; this one changed only how a question is answered.

AN UNKNOWN NAME IS A BETTER ANSWER THAN A PLAUSIBLE ONE.
THE CARD SHOWS A BADGE. NOBODY IS TOLD A LIE ABOUT WHO THEY ARE.

8 · What the night taught

  1. Repair at the stratum that owns the fault. Ninety-four duplicated rows became three hundred collisions two storeys up. The matcher hiding it was a repair attempted at the wrong altitude — and it worked, which is what made it dangerous.
  2. Measure before moving. Every claim made from reading code was wrong in some detail. The 12-of-33 key mismatch, the load race, the comma-subscript bug — all corrected by a probe.
  3. The captain’s objection is data. Retiring the ledger would have destroyed voice casting for a thousand figures. It was stopped by a question, not by an analysis.
  4. Simulate the migration before running it. The resolver shipped as a dry run that replayed every join both ways. It found one blocker, and the blocker was fixed before a single consumer changed.
  5. Adversarial tests, from real failures. Seventeen pairs, each an actual wrong answer taken from the live page, asserted on every load. Not configuration — a test suite that travels with the code.
  6. Silence is the enemy. A parse error that keeps serving, a fallback that swaps a thousand figures for thirty-eight without a word. Both worked as written; both cost hours.
  7. Rename nothing you do not have to. The whole repair fitted inside a lookup. Nothing downstream noticed, which is the definition of a migration done well.
